## §1 · The grimoire edit in PR #4 — rejected

PR #4 carries a hand-edit to `privacymage_grimoire_v10_1_0.json` (+12 / −4). It does three things:

- bumps `updated_at` from `2026-04-12` to `2026-04-17T22:41:23Z`;
- adds a new `meta.attributions[]` block crediting a *"bilateral_interface"* proverb to *Mitchell Travers + Christian Saucier* (the railroad/train framing — "methods need cloaking, and cloaking needs methods to interface with the world");
- adds a `bilateral_interface` entry to the proverb map and tidies two pieces of JSON formatting (`closing_chain` indentation, `key_additions` array wrapping).

**Disposition: REJECT — do not merge this file.** The content itself is not objectionable (the bilateral-interface framing is sound and worth preserving), but the *mechanism* is wrong. The grimoire is canonical and is regenerated/re-pinned through the established patch-and-pin pipeline, not amended inline in an application feature branch. Merging it here would fork the canonical artefact from its pinned source of truth.

> If the bilateral-interface attribution is worth keeping — and it reads like it is — it should re-enter the grimoire through the normal patch flow, with the IPFS CID updated accordingly. Carry the *idea* forward; drop the *diff*.

## §2 · PR #4 — the draft Archon integration (supersede, do not merge)

**Title:** *feat: Archon wallet integration — Mage identity, Vault save/restore, Runecraft proof fields*
**Branch:** `feature/archon-vault` · 7 commits · 17 files · +13,804 / −4,904

PR #4 was the first cut of the Archon wallet integration. Beyond the grimoire edit (§1), it carries a second, larger problem:

- **It destructively rewrites the canonical knowledge graph.** `src/data/nodes.ts` collapses from ~1,397 lines to 198, and `src/data/edges.ts` similarly — both replaced by an **auto-generated 18-node "Act-7 Boundary Blade decomposition"** demo dataset (header: *"AUTO-GENERATED by transmute-to-spellweb — DO NOT EDIT BY HAND … DID-Blind: true"*). This throws away the full 86-skill / 42-persona V10 graph and substitutes a single anonymised cloaking demo. That is a regression, not a feature.

The genuinely useful work in PR #4 — the Keymaster/Drawbridge wiring, the Archon-derived Mage identity, the vault save/restore, the Runecraft bilateral-proof payload fields — all reappears, cleaner, in PR #5.

**Disposition: CLOSE without merging, superseded by PR #5.** PR #4 should not be merged in any part: its grimoire edit is rejected (§1) and its graph-data rewrite is destructive. Everything worth keeping survives in PR #5.

## §3 · PR #5 — the Archon integration, rebased clean (accept)

**Title:** *feat: Archon wallet integration — Save Game, Vault backup/restore, PoH Cloak nodes*
**Branch:** `feature/archon-vault-v2` · 6 commits · 12 files · +12,686 / −1,979
**Commit 1 note:** *"Archon vault integration rebased onto v1.7.1 upstream."*

This is the same capability as PR #4, re-done correctly on top of current `main`. The decisive difference: **it touches neither the grimoire nor the canonical graph destructively.**

- `src/data/nodes.ts` / `edges.ts`: **+3 / +4 lines only** — it *appends* the PoH-cloak registry contribution rather than replacing the graph.
- No change to `privacymage_grimoire_v10_1_0.json`.

What it adds:

- **`src/contexts/KeymasterContext.tsx`** (new, ~625 lines) — a React context owning a module-level `DrawbridgeClient` + `CipherWeb`, wallet connect/export, and the vault verbs (`backupMageHistory`, `saveBladeToVault`, `listVaultItems` / `getVaultItem` / `deleteVaultItem`, plus the Weaver-vault read side). Gatekeeper URL defaults to `http://192.168.1.23:4224` (flaxlap.local, IP-direct because browsers bypass mDNS). A neat touch: it rebinds `gatekeeper.addText` to strip Drawbridge's JSON-quoted CIDs so vault item CIDs store as bare strings.
- **`src/lib/mageIdentity.ts`** (+~60 lines) — `deriveMageKeyFromKeymaster()`: signs a canonical inception document with the Archon secp256k1 DID key, then `SHA-256(sig)` → 32-byte Ed25519 seed → blade-signing key. Adds `archonLinked` / `archonDid` to `MageIdentity`; `initializeMageIdentity()` now accepts an optional `archonDid` and back-fills the link on existing identities. Overrides `@noble/ed25519`'s SHA-512 with a pure-JS impl for plain-HTTP contexts.
- **`src/lib/mageHistory.ts`** (new) — the `MageArchonBackup` v1 payload (identity, spells, constellations, forged blades, equipped blade, user edges, mana, and the optional `swordsmanLink` Excalibur binding) and the vault alias constants.
- **`src/data/poh-cloak-contribution.ts`** (new) — the 9-node / 6-edge PoH-cloak registry contribution (csaucier, 2026-05-11), *appended* to the graph rather than replacing it.
- **`src/components/SpellWeb.tsx`** (+458) — Archon vault state, Save-Game / backup-to-Archon, vault browser (list / load / delete), Weaver-vault import that injects registry nodes/edges into the live D3 graph, and auto-save of a blade to the vault on forge completion. Restore re-hydrates the `swordsmanLink` too.
- **`src/main.tsx`** — `Buffer` polyfill on `globalThis` + wraps the app in `KeymasterProvider`.
- **`src/utils/sessionPassphrase.ts`** (new) — in-memory session passphrase holder (never persisted — good).
- **`package.json`** — adds `@didcid/cipher`, `@didcid/gatekeeper`, `@didcid/keymaster`, `@noble/hashes`, `buffer`.

**Notes for the maintainer (non-blocking):**
- The default gatekeeper URL is a hard-coded LAN IP (`192.168.1.23:4224`). Fine for the bearer's bench; confirm it's overridable (it reads `spellweb-gatekeeper-url` from storage) before any public deploy, and that a plain-HTTP gatekeeper is intended (the SHA-512 override comment implies it is).
- `MageArchonBackup` uses `unknown[]` for the spell/constellation/blade arrays because the canonical types live inside `SpellWeb.tsx`. Documented and deliberate; the casts on restore are the trade-off.
- Worth a quick `tsc --noEmit` + `npm run build` gate before merge, given the new deps and the Buffer polyfill.

**Disposition: ACCEPT (the Archon integration of record).** Recommend a typecheck/build pass and a smoke test of connect → forge → auto-save → browse → restore before merging.
